# RAG FAQ Bot — Telegram-бот поддержки по вашей базе знаний

[English](README.md) | **Русский**

Telegram-бот, который отвечает клиентам **по вашей базе знаний** — загрузите FAQ,
прайс или документы, и бот отвечает по технологии RAG: находит подходящие фрагменты
и отвечает **с опорой на них и с указанием источника**, а не выдумывает. Если ответа
в ваших материалах нет — честно скажет об этом и направит в поддержку, а не
сгаллюцинирует.

Построен на aiogram 3 и работает на [**pgvector-rag**](https://github.com/xvin84/pgvector-rag) —
моей собственной open-source библиотеке для RAG — поверх PostgreSQL + pgvector.

## Возможности

**Для клиентов**
- Задайте вопрос обычным языком → получите ответ из базы знаний с названием
  источника.
- Нет нужной информации? Бот честно признаётся и предлагает обратиться в поддержку —
  ответ не выдумывается.

**Для владельца (админ)**
- Пришлите файл `.txt` / `.md` → бот разобьёт его на фрагменты, посчитает эмбеддинги
  и проиндексирует. Имя файла становится названием источника. Базу можно пополнять и
  менять в любой момент.

## Стек

Python 3.12 · aiogram 3 · [pgvector-rag](https://github.com/xvin84/pgvector-rag) ·
**PostgreSQL + pgvector** · OpenAI (эмбеддинги + чат) · asyncpg · pytest.

## Как это работает

1. **Индексация.** Загруженный документ режется на фрагменты (чанкинг по заголовкам),
   каждый эмбеддится и сохраняется в pgvector — всё через библиотеку `pgvector-rag`.
2. **Поиск.** Вопрос эмбеддится и сопоставляется с базой по косинусной близости;
   возвращаются только фрагменты выше порога релевантности.
3. **Заземление.** Найденные фрагменты становятся контекстом; модели предписано
   отвечать *только* по ним ([`services/answer.py`](services/answer.py)). Нет
   совпадений — фиксированный ответ «такой информации нет», и модель даже не
   вызывается.

Логика заземления и источников — чистые функции, покрытые тестами без OpenAI и базы.

## Запуск локально

```bash
docker compose up -d                 # PostgreSQL + pgvector
cp .env.example .env                 # указать BOT_TOKEN, ADMIN_IDS, OPENAI_API_KEY
uv sync
uv run python main.py
```

Затем как админ пришлите боту файл `.txt` или `.md` с вашим FAQ/документами и задавайте
вопросы. `OPENAI_BASE_URL` можно направить на прокси-агрегатор там, где
`api.openai.com` недоступен.

## Тесты

```bash
uv run pytest        # логика заземления и источников, без OpenAI и базы
```
